# GPT 批量邮箱管理与 Session 获取工具

这是一个本地运行的账号辅助管理工具，核心目标是解决两个实际需求：

1. 批量管理邮箱账号：集中导入、分组、检索、测试 IMAP、读取验证码、导出备份。
2. 批量登录 GPT 账号并获取 session：在自有账号和合法授权场景下，通过本地浏览器流程自动读取邮箱验证码，完成 GPT 登录，并将登录 session 保存在本地，方便统一管理、复制、导出和备份。

本项目适合用于个人或团队管理自己合法拥有的邮箱账号与 GPT 账号登录状态。项目默认只在本机运行，账号数据、数据库、浏览器配置、登录 session 等内容均保存在本地 `data/` 目录，不会主动上传到任何远程服务器。

请注意：session、Cookie、Token、邮箱密码、授权码都属于高度敏感数据。公开上传 GitHub 前，务必确认没有提交任何真实账号数据、数据库文件、日志文件、浏览器用户目录或导出的 session 文件。

## 项目定位

本项目不是在线平台，也不是账号服务。它是一个本地 Web 工具，运行后通过浏览器访问：

```text
http://127.0.0.1:8765/
```

主要用于在本地完成以下流程：

- 批量录入邮箱账号
- 批量检测邮箱 IMAP 是否可用
- 批量读取邮箱中的验证码
- 批量启动 GPT 登录任务
- 自动从邮箱中读取 GPT 登录验证码
- 登录成功后读取并保存 GPT session
- 批量复制或导出已保存的 session
- 对账号按分组、标签、状态进行管理

## 核心功能

### 1. 批量邮箱管理

邮箱管理是本项目的基础能力，适合处理大量邮箱账号的整理、检测和验证码读取工作。

支持功能包括：

- 手动新增、编辑、删除邮箱账号
- CSV 批量导入邮箱账号
- 导入前预览数据，减少错误导入
- 按邮箱、服务商、分组、状态等条件筛选账号
- 批量分组、批量删除、批量恢复分组
- 保存备注、标签、分组等管理信息
- 自动识别或填写常见邮箱服务商 IMAP 参数
- 单个或批量测试 IMAP 连接
- 单个或批量读取最新邮箱验证码
- 导出邮箱数据，支持脱敏导出和完整导出
- 记录操作日志和账号状态，方便排查失败原因

适合场景：

- 管理大量自有邮箱账号
- 检查邮箱授权码或 IMAP 是否可用
- 快速查找某个邮箱账号
- 按用途、客户、批次或状态对邮箱进行分组
- 统一读取邮箱验证码，提高本地测试效率

### 2. 批量 GPT 登录与 Session 获取

本项目支持在合法授权场景下，对自有 GPT 账号执行登录辅助流程，并在登录成功后保存 session。

基本流程如下：

1. 从邮箱列表中选择需要登录的账号。
2. 启动单个或批量 GPT 登录任务。
3. 工具打开本地浏览器访问 GPT 登录页面。
4. 自动填写邮箱地址并请求验证码。
5. 工具通过 IMAP 从对应邮箱中读取最新验证码。
6. 自动提交验证码并等待登录完成。
7. 登录成功后读取 GPT session。
8. session 保存到本地数据库，并可复制或导出。

批量登录支持：

- 选择多个邮箱账号后批量启动登录
- 设置并发数量，默认并发较保守
- 每个账号独立记录登录状态
- 展示成功、失败、重试、封禁或关闭等状态
- 登录成功后自动保存对应账号的 session
- 支持批量复制 session
- 支持导出 session JSON 文件

重要说明：

- 本功能仅适用于你自己合法拥有、合法使用、并被允许进行自动化辅助操作的账号。
- 如果目标平台禁止自动化登录、批量账号操作、session 导出或类似行为，请不要使用该功能。
- 本项目不提供绕过验证码、绕过风控、绕过安全验证、绕过平台限制或规避付费限制的能力。
- session 属于敏感登录凭据，泄露后可能导致账号被他人访问，请严格保管。

## 快速开始

### Windows 启动

双击运行：

```text
run-windows.bat
```

脚本会自动完成：

- 创建本地虚拟环境 `.venv/`
- 安装 Python 依赖
- 检查本地浏览器
- 必要时安装 Playwright Chromium
- 启动本地服务
- 打开浏览器访问管理页面

启动成功后访问：

```text
http://127.0.0.1:8765/
```

停止服务：

```text
stop-windows.bat
```

### macOS 启动

首次使用前添加执行权限：

```bash
chmod +x run-macos.command stop-macos.command
```

启动：

```bash
./run-macos.command
```

停止：

```bash
./stop-macos.command
```

### 手动启动

创建虚拟环境：

```bash
python -m venv .venv
```

```powershell

.\.venv\Scripts\python -m playwright install chromium
.\.venv\Scripts\python app.py
```

macOS / Linux:

```bash
./.venv/bin/python -m playwright install chromium
./.venv/bin/python app.py
```

浏览器访问：


## 使用流程

### 第一步：导入或新增邮箱

打开页面后，可以手动新增邮箱账号，也可以使用 CSV 批量导入。

建议为每个账号填写：

- 邮箱地址
- 邮箱密码、授权码或应用专用密码
- 邮箱服务商
- IMAP 地址
- IMAP 端口
- 分组
- 标签
- 备注

### 第二步：测试 IMAP

导入账号后，建议先使用「测试 IMAP」功能确认邮箱可连接。

如果测试失败，请检查：

- 邮箱地址是否正确
- 密码、授权码或应用专用密码是否正确
- 邮箱是否开启 IMAP
- IMAP 服务器和端口是否正确
- 网络、防火墙或代理是否阻止连接

### 第三步：读取验证码

IMAP 测试通过后，可以使用「读取验证码」功能读取邮箱中最新的一次性验证码。

批量读取验证码适合在本地测试或账号整理时使用。请只读取你自己有权访问的邮箱内容。

### 第四步：批量 GPT 登录

选择需要登录的邮箱账号，启动批量 GPT 登录任务。

工具会按设置的并发数量执行登录流程：

- 打开 GPT 登录页面
- 填写邮箱
- 请求邮箱验证码
- 从邮箱读取验证码
- 提交验证码
- 等待登录成功
- 读取并保存 session

每个账号会显示独立状态，方便判断哪些账号成功、哪些账号需要重试。

### 第五步：复制或导出 Session

登录成功后，可以复制单个账号的 session，也可以批量复制或导出 session。

session 导出文件应只保存在安全位置，不应上传到 GitHub、网盘、群聊、论坛或任何公开平台。

## CSV 导入格式

CSV 可参考以下格式：

```csv
email,password,provider,imap_host,imap_port,group_name,tags,remark
test@example.com,app-password,outlook,imap-mail.outlook.com,993,group-a,"tag1,tag2",demo account
```

字段说明：

- `email`：邮箱地址
- `password`：邮箱密码、授权码或应用专用密码
- `provider`：邮箱服务商标识
- `imap_host`：IMAP 服务器地址
- `imap_port`：IMAP 端口，通常为 `993`
- `group_name`：分组名称
- `tags`：标签
- `remark`：备注

请不要将包含真实账号密码的 CSV 上传到公开仓库。

## Session 数据说明

GPT session 是登录状态数据，通常可能包含 Cookie、Token、用户状态或其他认证信息。拥有 session 的人可能在一定条件下访问对应账号，因此它必须按照敏感凭据处理。

请遵守以下原则：

- 只获取和保存自己合法拥有账号的 session
- 不要导出、出售、共享或传播他人 session
- 不要把 session 提交到 GitHub
- 不要把 session 写入 README、Issue、提交记录或日志截图
- 定期清理不需要的 session
- 如果怀疑 session 泄露，请立即退出登录、修改密码并重新生成登录状态

## 常见问题

### 为什么 IMAP 连接失败？

常见原因包括：

- 邮箱没有开启 IMAP
- 使用了普通密码而不是应用专用密码
- 授权码过期或填写错误
- IMAP 地址或端口不正确
- 邮箱服务商限制了登录来源
- 本机网络、防火墙或代理阻止了连接

### 为什么 GPT 登录失败？

常见原因包括：

- 网络无法访问 GPT 登录页面
- 账号需要额外安全验证
- 邮箱验证码没有及时收到
- 验证码邮件被归入垃圾箱或延迟送达
- 平台页面结构更新导致自动化流程失效
- 账号状态异常或被平台限制
- 自动化登录行为不符合目标平台规则

### 为什么没有读取到验证码？

可能原因包括：

- 邮箱 IMAP 不可用
- 验证码邮件尚未送达
- 验证码邮件标题或发件人发生变化
- 邮件被归档、过滤或移动到其他文件夹
- 邮箱服务商限制了 IMAP 读取
